Preguntas a Responder

    ¿Por qué es una buena práctica separar el controlador de la vista? - Para obtener un codigo mejor estructurado y ordenado. Facilita la búsqueda de errores y ubicación de partes de código que se busquen mejorar.

    Aparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?

      Superglobals - Superglobals son variables internas que están disponibles siempre en todos los ámbitos.
      $Globals - Hace referencia a todas las variables disponibles en el ámbito global.
      $_SERVER - Información del entorno del servidor y de ejecución.
      $_FILES - Variables de subida de ficheros HTTP.
      $_REQUEST - Variables HTTP Request.
      $_SESSION - Variables de sesión.
      $_ENV - Variables de entorno.
      $_COOKIE - Cookies HTTP.
      $php_errormsg - El mensaje de error anterior.
      $HTTP_RAW_POST_DATA - Datos POST sin tratar.
      $http_response_header - Encabezados de respuesta HTTP.
      $argc -El número de argumentos pasados a un script.
      $argv -Array de argumentos pasados a un script. [1]

    Explora las funciones de php, y describe 2 que no hayas visto en otro lenguaje y que llamen tu atención.

      soundex() - Calcula la clave soundex de un string. Las claves soundex tienen la propiedad que palabras pronunciadas de forma similar producen la misma clave soundex y por lo tanto puede ser usada para simplificar búsquedas en bases de datos donde se conoce la pronunciación pero no la ortografía. [2]
      taint() - Corrompe un string. Crea un string corrompido. Solamente se usa para realizar pruebas. [3]

    ¿Qué es XSS y cómo se puede prevenir? - "Cross-Site Scripting" son ataques en forma de entrada, en los cuales scripts maliciosos son insertados en sitios web confiables. Estos ataques ocurren cuando un atacante usa un aaplicación web para mandar codigo malicioso, generalmente en la forma de un "side-script" de parte del sitio hacia un usuario [4]. Pueden ser prevenidos siguiendo las siguientes reglas:

    • HTML Encode Before Inserting Untrusted Data into HTML Element Content
    • Attribute Encode Before Inserting Untrusted Data into HTML Common Attributes
    • JavaScript Encode Before Inserting Untrusted Data into JavaScript Data Values
    • URL Encode Before Inserting Untrusted Data into HTML URL Parameter Values
    • Sanitize HTML Markup with a Library Designed for the Job
    • Avoid JavaScript URLs
    • Prevent DOM-based XSS [5]
