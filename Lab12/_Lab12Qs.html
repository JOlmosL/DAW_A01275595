Preguntas a Responder

    ¿Por qué es importante hacer un session_unset() y luego un session_destroy()? - Porque el session_unset() se utiliza para eliminar todos los pares de nombre / valor de sesión de los datos de sesión, manteniendo la cookie de sesión de ID de sesión y al utilizar session_destroy() se eliminaran todos los pares de nombre / valor de sesión asociados con la sesión, así como el valor de ID de sesión asignado a la cookie de sesión del navegador del cliente. [1]

    ¿Cuál es la diferencia entre una variable de sesión y una cookie?
    Variable de sesión Sirven para controlar el acceso a una página web de una zona privada
    Cookies Es un fragmento de información que un navegador web almacena en el disco duro del visitante a una página web [2]

    ¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?Se hace una revisión de las variables de sesión y en caso de que ya existan con el mismo nombre, se le notifica al usuario.

    ¿Qué es CSRF y cómo puede prevenirse?En inglés, Cross-Site Request Forgery es un ataque que fuerza a un end-user a ejecutar acciones no deseadas en una aplicación web en la cual estan actualmente autenticados [3]. Entre las acciones que podemos hacer para prevenirlas, recomiendo:

    1. Checar que el framework haya sido construido con protección CSRF.
    2. Siempre usar SameSite Cookie Atribute para cookies de sesión.
    3. Implementar una forma de mitigación.
    4. Considerar la implementación de protección basada en la interacción del usuario para operaciones altamente sensibles.
    5. XSS puede usarse derrotar cualquier tecnica de mitigación CSRF. [4]
